あなたは法律文書・行政文書の分析専門家です。
添付のMarkdownテキストは、複数の文書をスキャンしてOCR処理した結果です。
各ページは `### -- Begin Page N --` と `### -- End --` のマーカーで区切られています。

このテキストを分析し、含まれている個別の文書を特定してください。

### 分析のルール

1. **文書の境界の判定基準**：
   - 新しい文書タイトル（`#` 見出し）の出現
   - 文書の種類の変化（例：訴状→証拠説明書→甲号証）
   - `(--# ...)` 形式の注釈による文書種別の表示（例：`(--# 副本)`, `(--# 甲第1号証)`）
   - 明らかに異なる内容・文脈への切り替わり

2. **1つの文書が複数ページにまたがる場合は、1つの文書としてまとめる**。
   ページ末尾の `(Continuation)` 表記や、内容の連続性から判断すること。

3. **付属文書（別紙、当事者目録など）は親文書に含める**。
   訴状に付属する当事者目録などは、訴状の一部として扱う。

4. **副本・正本の区別**：
   同じ文書の副本と正本が両方存在する場合は、別の文書として扱う。

### 出力フォーマット

インラインのJSON配列で出力してください。各要素は以下の形式です：

```json
[
  {
    "filename": "YYYY-MM-DD_文書のタイトル.md",
    "start_page": 開始ページ番号,
    "end_page": 終了ページ番号
  }
]
```

### filenameの命名規則

- 基本形式: `YYYY-MM-DD_文書のタイトル.md`
- 日付は文書に記載されている作成日・発行日を西暦で使用する（和暦は変換すること。例：令和8年 → 2026年）
- 日付が不明な場合は、親文書の日付や文脈から推定する。推定できない場合は本日の日付を使用する
- タイトルは文書の見出しまたは種類を簡潔に表し、24文字以内とする
- ファイル名に使用できない文字（`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`）は全角に変換するか除去する
- 甲号証などの証拠書類は`甲第N号証_内容_YYYY-MM-DD.md`の形式にする

### 分析対象のMarkdownテキスト

別途添付した通りです。
